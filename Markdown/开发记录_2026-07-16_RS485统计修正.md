# 2026年7月16日 RS485主机统计修正开发记录

## 问题现象

- 主机挂载两个从机做 RS485 通讯时，只使能 UART1 或 UART3 中一个，界面上两个串口的发送统计都会变化。
- 实际主机 485 轮询使用一条物理总线，现象主要来自统计口径混用。

## 原因分析

- 当前 RS485 主机物理发送口统一为 `RS422_PORT_U9`。
- UART1 / UART3 在 RS485 模式下是两个逻辑测试实例，分别对应从机地址 1 和地址 2。
- 修改前 `RS485_SpeedTest_GetStats()` 把 `RS422_PORT_U9` 的物理总线 TX/RX 总量同时返回给两个逻辑实例，导致未使能的一路也显示出发送/接收数据。

## 修改内容

- 在 `RS485_SpeedTest_GetStats()` 中增加未使能逻辑从机统计归零处理。
- UART1 / UART3 的 `tx_bytes` 改为由各自从机的 `tx_frames * RS422_PROTOCOL_FRAME_LENGTH` 计算。
- UART1 / UART3 的 `rx_bytes` 改为由各自从机的 `rx_frames * RS422_PROTOCOL_FRAME_LENGTH` 计算。
- 增加 `Tests/host_comm_rs485_disabled_stats_zero_check.ps1`，防止后续再次把共享物理总线统计误报给两个逻辑串口。

## 验证结果

- `Tests/host_comm_rs485_disabled_stats_zero_check.ps1`：通过。
- `Tests/rs485_master_address_poll_check.ps1`：通过。
- `Tests/host_comm_protocol_check.ps1`：通过。
- Keil 编译：`0 Error(s), 0 Warning(s)`。

## 当前结论

只使能 UART1 时，只会显示从机地址 1 的统计；只使能 UART3 时，只会显示从机地址 2 的统计。另一路未使能时统计保持 0。
